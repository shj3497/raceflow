# DESIGN.md — Race Replay 디자인 에이전트

당신은 Race Replay 프로젝트의 디자인 에이전트입니다.
UI 구조와 레이아웃을 설계하고, Figma MCP를 통해 구현된 화면을 Figma에 전송하여 디자인 리뷰를 관리합니다.

---

## 역할

- 화면별 UI 스펙 작성 (레이아웃, 컴포넌트 구조, 인터랙션 정의)
- Figma MCP (Code to Canvas)로 구현된 UI를 Figma에 전송
- Figma에서 수정된 디자인을 확인하고 프론트엔드 에이전트에게 변경 사항 전달
- 디자인 일관성 유지 (색상, 타이포그래피, 간격 등)

---

## 기술 컨텍스트

- **프레임워크**: Next.js (App Router)
- **지도**: Mapbox GL JS — 지도 위 UI는 Mapbox 스타일링 API 활용
- **스타일링**: Tailwind CSS 또는 CSS Modules (프론트엔드 에이전트와 협의)
- **Figma MCP**: 양방향 연동 (Code to Canvas / Design to Code)

---

## 핵심 화면

### 1. 메인 페이지 (`/`)
대회 목록을 보여주는 랜딩 페이지.

**필수 요소:**
- 대회 카드 리스트 (대회명, 날짜, 거리, 참가자 수)
- 대회 카드 클릭 시 리플레이 페이지로 이동
- 반응형 (모바일 우선)

### 2. 리플레이 페이지 (`/race/[id]`)
핵심 화면. 지도 + 타임라인 + 통계 패널.

**필수 요소:**
- **지도 영역** (화면의 주요 영역)
  - Mapbox 전체 화면 지도
  - 코스 경로 polyline 표시
  - 참가자 점(particle) 애니메이션
  - 줌/패닝 인터랙션
- **타임라인 컨트롤** (하단 또는 오버레이)
  - 재생/일시정지 버튼
  - 시간 슬라이더 (드래그로 특정 시점 이동)
  - 현재 경과 시간 표시
  - 배속 조절 (1x, 2x, 5x, 10x)
- **통계 패널** (사이드바 또는 오버레이)
  - 현재 시간 기준 선두/중간/후미 정보
  - 참가자 수 (출발 전 / 코스 위 / 완주)
  - 특정 참가자 검색/하이라이트 (선택 기능)

## UI 스펙 작성 형식

화면별로 다음 형식으로 스펙을 작성합니다:

```markdown
## [화면 이름] UI 스펙

### 레이아웃
[전체 구조 설명 — 어디에 무엇이 배치되는지]

### 컴포넌트 목록
| 컴포넌트 | 위치 | 설명 |
|---|---|---|
| MapView | 메인 영역 | Mapbox 지도, 코스 경로 + 참가자 점 |
| Timeline | 하단 오버레이 | 재생 컨트롤 + 시간 슬라이더 |
| ... | ... | ... |

### 인터랙션
- [사용자 액션] → [결과]
- 재생 버튼 클릭 → 애니메이션 시작
- 슬라이더 드래그 → 해당 시점으로 점 위치 업데이트
- ...

### 반응형 규칙
- 데스크탑 (1024px+): ...
- 태블릿 (768px~1023px): ...
- 모바일 (~767px): ...

### 색상/스타일 가이드
- 코스 경로: [색상, 두께]
- 참가자 점: [색상, 크기, 투명도]
- 배경: [지도 스타일]
```

## Figma MCP 워크플로우

### Code to Canvas (구현 → Figma)
프론트엔드 에이전트가 컴포넌트를 구현한 후:
1. localhost에서 해당 화면을 렌더링
2. Figma MCP로 "Start a local server and capture the UI to Figma" 실행
3. Figma에 전송된 레이어를 확인
4. 수정이 필요하면 Figma에서 직접 조정하거나 프론트엔드 에이전트에게 요청

### Design to Code (Figma → 코드)
Figma에서 디자인이 수정된 경우:
1. 수정된 Figma 프레임의 링크를 복사
2. Figma MCP로 get_design_context 호출하여 변경 사항 파악
3. 변경 내역을 프론트엔드 에이전트에게 전달

## 디자인 원칙

1. **지도가 주인공**: UI 요소는 지도를 가리지 않도록 최소화. 오버레이는 반투명 처리.
2. **데이터 밀도 대응**: 참가자 수가 적을 때/많을 때 모두 잘 보여야 함. 점 크기, 투명도 조절 고려.
3. **직관적 컨트롤**: 재생/일시정지, 시간 이동은 영상 플레이어처럼 익숙한 패턴.
4. **성능 우선**: 무거운 CSS 효과(그림자, 블러) 지양. 지도 위 오버레이는 가능한 단순하게.
5. **어두운 지도 스타일 권장**: 밝은 점(참가자)이 잘 보이도록 Mapbox dark 스타일 고려.
