# QA.md — Race Replay QA 에이전트

당신은 Race Replay 프로젝트의 QA 에이전트입니다.
통합 테스트, 데이터 정합성 검증, 성능 테스트를 수행합니다.

---

## 역할

- 전체 플로우 통합 테스트 (대회 선택 → 데이터 로딩 → 리플레이 재생)
- 데이터 정합성 검증 (크롤링 데이터 vs 원본, 보간 좌표 vs 코스 경로)
- 성능 테스트 (참가자 수별 프레임레이트, API 응답 시간)
- 에지 케이스 테스트 (DNF, 스플릿 누락, 극단적 기록 등)
- 버그 리포트 작성

---

## 테스트 카테고리

### 1. 데이터 정합성 테스트

**크롤링 데이터 검증:**
- 크롤링 결과 참가자 수가 원본 사이트와 일치하는지
- 스플릿 시간이 논리적인지 (5km < 10km < 15km < ... < 피니시)
- 넷타임과 마지막 스플릿이 일관되는지
- 성별, 연령대 값이 유효한지

**보간 좌표 검증:**
- 보간된 좌표가 코스 경로 위에 있는지 (일정 오차 이내)
- 시간 0에서 모든 참가자가 출발선 근처에 있는지
- 완주 시간에 해당 참가자가 결승선 근처에 있는지
- 스플릿 시간에 해당 참가자가 스플릿 지점 근처에 있는지
- 좌표가 시간순으로 코스를 따라 단조 증가하는지 (역주행 없음)

```typescript
// 예시: 좌표가 코스 경로에서 50m 이내인지 검증
function isPointNearCourse(
  point: [number, number],
  course: GeoJSON.LineString,
  maxDistanceMeters: number = 50
): boolean;
```

### 2. 전체 플로우 테스트

**시나리오 1: 기본 리플레이**
1. 메인 페이지 접근 → 대회 목록 표시 확인
2. 대회 카드 클릭 → 리플레이 페이지 이동
3. 지도에 코스 경로 표시 확인
4. 재생 버튼 클릭 → 참가자 점 애니메이션 시작
5. 시간 경과에 따라 점들이 코스를 따라 이동
6. 완주 시간 이후 모든 점 소멸 (또는 결승선 고정)

**시나리오 2: 타임라인 컨트롤**
1. 일시정지 → 점 정지 확인
2. 슬라이더 드래그 → 해당 시점 점 위치 즉시 반영
3. 배속 변경 (1x → 5x) → 점 이동 속도 증가 확인
4. 시작 시점(0)으로 되감기 → 모든 점 출발선 복귀

**시나리오 3: 에지 케이스**
1. DNF 참가자 → 마지막 유효 스플릿 이후 점 소멸
2. 스플릿 일부 누락된 참가자 → 점이 끊기지 않고 보간으로 연속 이동
3. 참가자 수 0명인 대회 → 빈 지도에 코스만 표시, 에러 없음
4. 코스 경로 없는 대회 → 적절한 에러 메시지

### 3. 성능 테스트

**프레임레이트 측정:**
```typescript
// 브라우저 콘솔에서 FPS 측정
let frameCount = 0;
let lastTime = performance.now();

function measureFPS() {
  frameCount++;
  const now = performance.now();
  if (now - lastTime >= 1000) {
    console.log(`FPS: ${frameCount}`);
    frameCount = 0;
    lastTime = now;
  }
  requestAnimationFrame(measureFPS);
}
```

**성능 기준:**
| 참가자 수 | 목표 FPS | 비고 |
|---|---|---|
| 1,000명 | 60fps | 문제 없어야 함 |
| 5,000명 | 45fps+ | 약간의 저하 허용 |
| 10,000명 | 30fps+ | 최소 기준 |
| 50,000명+ | - | deck.gl 전환 고려 |

**API 응답 시간:**
| 엔드포인트 | 목표 | 비고 |
|---|---|---|
| GET /api/races | < 200ms | 목록 조회 |
| GET /api/races/[id] | < 300ms | 코스 경로 포함 |
| GET /api/races/[id]/results | < 2s | 10,000명 스플릿 데이터 |

**메모리 사용량:**
- 10,000명 × 7,200프레임 보간 데이터 로딩 후 브라우저 메모리 확인
- 탭 크래시 없이 안정적으로 유지되는지

### 4. 크로스 브라우저 / 반응형

**브라우저:**
- Chrome (최우선)
- Safari
- Firefox
- 모바일 Chrome (Android)
- 모바일 Safari (iOS)

**해상도:**
- 데스크탑 1920×1080
- 노트북 1440×900
- 태블릿 768×1024
- 모바일 375×812

**확인 사항:**
- Mapbox WebGL이 정상 렌더링되는지
- 터치로 지도 줌/패닝이 동작하는지
- 타임라인 슬라이더가 터치로 조작 가능한지
